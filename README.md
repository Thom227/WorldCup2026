# WK 2026 Poule

Friend-group World Cup 2026 prediction game. Participants fill in the form and submit — their predictions arrive automatically as JSON via Web3Forms. The admin saves the JSON, runs a build script, and pushes — the dashboard updates automatically.

## Local dev

```bash
python3 -m http.server 8080
```

| URL | Purpose |
|-----|---------|
| `http://localhost:8080` | Prediction form |
| `http://localhost:8080/dashboard.html` | Scoring dashboard |
| `http://localhost:8080/?dev` | Form with test-data tools |

## Adding a participant

Submissions arrive automatically in your inbox via Web3Forms. The email contains the full JSON in the `submission` field.

1. Copy the JSON from the email, save to `submissions/<naam>.json` (local only, never committed)
2. `node build.js`
3. Commit and push `data/dashboard-data.json`

> If a participant only sent a PDF (no JSON in email), use `docs/extractie-prompt.md` to extract it.

## Updating results during the tournament

After each match day, edit `data/results.json` and run the build:

```bash
node build.js
# then commit and push:
# git add data/results.json data/dashboard-data.json
```

### Group stage (11 jun – 28 jun)

Add played matches to `groupScores`. Key = match index (0–71), value = `{"h": 2, "a": 1}`.
Set `"phase": "group"` on the first update.

```json
"groupScores": {
  "0": {"h": 2, "a": 1},
  "1": {"h": 0, "a": 2}
}
```

### After group stage ends

Fill `groupStandings` with the actual finishing order per group. **Required** — the KO scoring needs actual team names.
Set `"phase": "knockout"`.

```json
"groupStandings": {
  "A": ["Mexico", "Tsjechië", "Zuid-Afrika", "Zuid-Korea"],
  "B": ["Canada", "Zwitserland", "Qatar", "Bosnië-Herzegovina"]
}
```

### Knockout phase (28 jun – 19 jul)

Add each played KO match to `ko`. Key = match ID, value = actual teams + scores.

```json
"ko": {
  "r32_0": {"h": "Mexico", "a": "Tsjechië", "sh": 2, "sa": 0},
  "af_0":  {"h": "Mexico", "a": "Nederland", "sh": 1, "sa": 2}
}
```

KO match IDs: `r32_0`–`r32_15`, `af_0`–`af_7`, `kf_0`–`kf_3`, `hf_0`, `hf_1`, `tf`, `fin`

### After the tournament

Fill `bonus` answers in `data/results.json` and set `"phase": "done"`.

```json
"bonus": {
  "b1": "Memphis Depay",
  "b2": "Brazilië",
  "b3": "Iran",
  "b4": 3,
  "b5": "Kylian Mbappé",
  "b6": 4,
  "b7": "Nederland - Japan",
  "b8": "Argentinië",
  "b9": 142
}
```

## Files

| File | Purpose |
|------|---------|
| `index.html` / `app.js` / `styles.css` | Prediction form |
| `dashboard.html` | Live leaderboard |
| `build.js` | Generates `data/dashboard-data.json` from local submissions + results |
| `data/matches.json` | Fixture schedule — never edit |
| `data/results.json` | Actual results — you edit this during the tournament |
| `data/dashboard-data.json` | Generated output — always commit after running build.js |
| `vendor/` | Bundled third-party libraries |
| `assets/` | Images and other static assets |
| `submissions/` | Local only, gitignored — one JSON per participant + readable `.txt` generated by build.js |
| `docs/extractie-prompt.md` | AI prompt to extract JSON from a participant's PDF |
